.. _details-d4/d46/classcv_1_1___output_array:

Detailed Documentation
~~~~~~~~~~~~~~~~~~~~~~

This type is very similar to InputArray except that it is used for input/output and output function parameters.

Just like with InputArray, OpenCV users should not care about OutputArray, they just pass ``:ref:`Mat <doxid-db/de6/classcv_1_1_mat>```, ``vector<T>`` etc. to the functions. The same limitation as for ``InputArray`` : *Do not explicitly create OutputArray instances* applies here too.

If you want to make your function polymorphic (i.e. accept different arrays as output parameters), it is also not very difficult. Take the sample above as the reference. Note that :ref:`_OutputArray::create() <doxid-d4/d46/classcv_1_1___output_array_1ae85544adf2eaeae547c742b5031e0cc5>` needs to be called before :ref:`_OutputArray::getMat() <doxid-d6/d3a/classcv_1_1___input_array_1a9c09739ca3e0ce212e2ab8374aa2f195>`. This way you guarantee that the output array is properly allocated.

Optional output parameters. If you do not need certain output array to be computed and returned to you, pass :ref:`cv::noArray() <doxid-dc/d84/group__core__basic_1gad9287b23bba2fed753b36ef561ae7346>`, just like you would in the case of optional input array. At the implementation level, use :ref:`_OutputArray::needed() <doxid-d4/d46/classcv_1_1___output_array_1a2ef2667f36017be9379d95fc5f421976>` to check if certain output array needs to be computed or not.

There are several synonyms for OutputArray that are used to assist automatic Python/Java/... wrapper generators:

.. ref-code-block:: cpp

	typedef OutputArray OutputArrayOfArrays;
	typedef OutputArray InputOutputArray;
	typedef OutputArray InputOutputArrayOfArrays;
