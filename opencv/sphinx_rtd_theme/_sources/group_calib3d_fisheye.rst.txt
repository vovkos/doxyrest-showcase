Detailed Documentation
~~~~~~~~~~~~~~~~~~~~~~

Definitions: Let P be a point in 3D of coordinates X in the world reference frame (stored in the matrix X) The coordinate vector of P in the camera reference frame is:

.. math::

	Xc = R X + T

where R is the rotation matrix corresponding to the rotation vector om: R = rodrigues(om); call x, y and z the 3 coordinates of Xc:

.. math::

	x = Xc_1 \\ y = Xc_2 \\ z = Xc_3

The pinhole projection coordinates of P is [a; b] where

.. math::

	a = x / z \ and \ b = y / z \\ r^2 = a^2 + b^2 \\ \theta = atan(r)

Fisheye distortion:

.. math::

	\theta_d = \theta (1 + k_1 \theta^2 + k_2 \theta^4 + k_3 \theta^6 + k_4 \theta^8)

The distorted point coordinates are [x'; y'] where

.. math::

	x' = (\theta_d / r) a \\ y' = (\theta_d / r) b

Finally, conversion into pixel coordinates: The final pixel coordinates vector [u; v] where:

.. math::

	u = f_x (x' + \alpha y') + c_x \\ v = f_y y' + c_y

Performs camera calibaration.



.. rubric:: Parameters:

.. list-table::
	:widths: 20 80

	*
		- objectPoints

		- vector of vectors of calibration pattern points in the calibration pattern coordinate space.

	*
		- imagePoints

		- vector of vectors of the projections of calibration pattern points. imagePoints.size() and objectPoints.size() and imagePoints[i].size() must be equal to objectPoints[i].size() for each i.

	*
		- image_size

		- Size of the image used only to initialize the intrinsic camera matrix.

	*
		- K

		- Output 3x3 floating-point camera matrix :math:`A = \vecthreethree{f_x}{0}{c_x}{0}{f_y}{c_y}{0}{0}{1}`. If :ref:`fisheye::CALIB_USE_INTRINSIC_GUESS <doxid-db/d58/group__calib3d__fisheye_1ggaeb8b145b1fc476e4e97d3b332447b24ca0899eaa2f96d6eed9927c4b4f4464e05>` / is specified, some or all of fx, fy, cx, cy must be initialized before calling the function.

	*
		- D

		- Output vector of distortion coefficients :math:`(k_1, k_2, k_3, k_4)`.

	*
		- rvecs

		- Output vector of rotation vectors (see Rodrigues ) estimated for each pattern view. That is, each k-th rotation vector together with the corresponding k-th translation vector (see the next output parameter description) brings the calibration pattern from the model coordinate space (in which object points are specified) to the world coordinate space, that is, a real position of the calibration pattern in the k-th pattern view (k=0.. *M* -1).

	*
		- tvecs

		- Output vector of translation vectors estimated for each pattern view.

	*
		- flags

		- 
		  Different flags that may be zero or a combination of the following values:
		  
		  * :ref:`fisheye::CALIB_USE_INTRINSIC_GUESS <doxid-db/d58/group__calib3d__fisheye_1ggaeb8b145b1fc476e4e97d3b332447b24ca0899eaa2f96d6eed9927c4b4f4464e05>` cameraMatrix contains valid initial values of fx, fy, cx, cy that are optimized further. Transforms an image to compensate for fisheye lens distortion.

The function is simply a combination of :ref:`fisheye::initUndistortRectifyMap <doxid-db/d58/group__calib3d__fisheye_1ga0d37b45f780b32f63ed19c21aa9fd333>` (with unity R ) and remap (with bilinear interpolation). See the former function for details of the transformation being performed.

See below the results of undistortImage.

* a) result of undistort of perspective camera model (all possible coefficients (k_1, k_2, k_3, k_4, k_5, k_6) of distortion were optimized under calibration)
  
  * b) result of :ref:`fisheye::undistortImage <doxid-db/d58/group__calib3d__fisheye_1ga167df4b00a6fd55287ba829fbf9913b9>` of fisheye camera model (all possible coefficients (k_1, k_2, k_3, k_4) of fisheye distortion were optimized under calibration)
  
  * c) original image was captured with fisheye lens

Pictures a) and b) almost the same. But if we consider points of image located far from the center of image, we can notice that on image a) these points are distorted.
