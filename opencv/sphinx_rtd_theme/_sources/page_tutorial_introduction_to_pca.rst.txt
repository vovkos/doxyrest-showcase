Introduction to Principal Component Analysis (PCA)
==================================================

.. rubric:: Goal

In this tutorial you will learn how to:

* Use the OpenCV class :ref:`cv::PCA <doxid-d3/d39/classcv_1_1_p_c_a>` to calculate the orientation of an object.

.. rubric:: What is PCA?

Principal Component Analysis (PCA) is a statistical procedure that extracts the most important features of a dataset.

.. image:: pca_line.png

Consider that you have a set of 2D points as it is shown in the figure above. Each dimension corresponds to a feature you are interested in. Here some could argue that the points are set in a random order. However, if you have a better look you will see that there is a linear pattern (indicated by the blue line) which is hard to dismiss. A key point of PCA is the Dimensionality Reduction. Dimensionality Reduction is the process of reducing the number of the dimensions of the given dataset. For example, in the above case it is possible to approximate the set of points to a single line and therefore, reduce the dimensionality of the given points from 2D to 1D.

Moreover, you could also see that the points vary the most along the blue line, more than they vary along the Feature 1 or Feature 2 axes. This means that if you know the position of a point along the blue line you have more information about the point than if you only knew where it was on Feature 1 axis or Feature 2 axis.

Hence, PCA allows us to find the direction along which our data varies the most. In fact, the result of running PCA on the set of points in the diagram consist of 2 vectors called *eigenvectors* which are the *principal components* of the data set.

.. image:: pca_eigen.png

The size of each eigenvector is encoded in the corresponding eigenvalue and indicates how much the data vary along the principal component. The beginning of the eigenvectors is the center of all points in the data set. Applying PCA to N-dimensional data set yields N N-dimensional eigenvectors, N eigenvalues and 1 N-dimensional center point. Enough theory, let’s see how we can put these ideas into code.

.. rubric:: How are the eigenvectors and eigenvalues computed?

The goal is to transform a given data set **X** of dimension *p* to an alternative data set **Y** of smaller dimension *L*. Equivalently, we are seeking to find the matrix **Y**, where **Y** is the *Karhunen–Loève transform* (KLT) of matrix **X** :

.. math::

	\mathbf{Y} = \mathbb{K} \mathbb{L} \mathbb{T} \{\mathbf{X}\}

**Organize the data set**

Suppose you have data comprising a set of observations of *p* variables, and you want to reduce the data so that each observation can be described with only *L* variables, *L* <*p*. Suppose further, that the data are arranged as a set of *n* data vectors :math:`x_1...x_n` with each :math:`x_i` representing a single grouped observation of the *p* variables.

* Write :math:`x_1...x_n` as row vectors, each of which has *p* columns.

* Place the row vectors into a single matrix **X** of dimensions :math:`n\times p`.

**Calculate the empirical mean**

* Find the empirical mean along each dimension :math:`j = 1, ..., p`.

* Place the calculated mean values into an empirical mean vector **u** of dimensions :math:`p\times 1`.
  
  .. math::
  
  	\mathbf{u[j]} = \frac{1}{n}\sum_{i=1}^{n}\mathbf{X[i,j]}

**Calculate the deviations from the mean**

Mean subtraction is an integral part of the solution towards finding a principal component basis that minimizes the mean square error of approximating the data. Hence, we proceed by centering the data as follows:

* Subtract the empirical mean vector **u** from each row of the data matrix **X**.

* Store mean-subtracted data in the :math:`n\times p` matrix **B**.
  
  .. math::
  
  	\mathbf{B} = \mathbf{X} - \mathbf{h}\mathbf{u^{T}}
  
  where **h** is an :math:`n\times 1` column vector of all 1s:
  
  .. math::
  
  	h[i] = 1, i = 1, ..., n

**Find the covariance matrix**

* Find the :math:`p\times p` empirical covariance matrix **C** from the outer product of matrix **B** with itself:
  
  .. math::
  
  	\mathbf{C} = \frac{1}{n-1} \mathbf{B^{*}} \cdot \mathbf{B}
  
  where \* is the conjugate transpose operator. Note that if B consists entirely of real numbers, which is the case in many applications, the "conjugate transpose" is the same as the regular transpose.

**Find the eigenvectors and eigenvalues of the covariance matrix**

* Compute the matrix **V** of eigenvectors which diagonalizes the covariance matrix **C** :
  
  .. math::
  
  	\mathbf{V^{-1}} \mathbf{C} \mathbf{V} = \mathbf{D}
  
  where **D** is the diagonal matrix of eigenvalues of **C**.

* Matrix **D** will take the form of an :math:`p \times p` diagonal matrix:
  
  .. math::
  
  	D[k,l] = \left\{\begin{matrix} \lambda_k, k = l \\ 0, k \neq l \end{matrix}\right.
  
  here, :math:`\lambda_j` is the *j* -th eigenvalue of the covariance matrix **C**

* Matrix **V**, also of dimension *p* x *p*, contains *p* column vectors, each of length *p*, which represent the *p* eigenvectors of the covariance matrix **C**.

* The eigenvalues and eigenvectors are ordered and paired. The *j* th eigenvalue corresponds to the *j* th eigenvector.

	double getOrientation(const vector<Point> &pts, :ref:`Mat <doxid-db/de6/classcv_1_1_mat>` &img)
	{
	    //Construct a buffer used by the pca analysis
	    int sz = static_cast<int>(pts.size());
	    :ref:`Mat <doxid-db/de6/classcv_1_1_mat>` data_pts = :ref:`Mat <doxid-db/de6/classcv_1_1_mat>`(sz, 2, :ref:`CV_64FC1 <doxid-d1/d1b/group__core__hal__interface_1ga3b256d68c6002b6afd60ab9538d717a8>`);
	    for (int i = 0; i < data_pts.:ref:`rows <doxid-db/de6/classcv_1_1_mat_1abed816466c45234254d25bc59c31245e>`; ++i)
	    {
	        data_pts.:ref:`at <doxid-db/de6/classcv_1_1_mat_1aa5d20fc86d41d59e4d71ae93daee9726>`<double>(i, 0) = pts[i].x;
	        data_pts.:ref:`at <doxid-db/de6/classcv_1_1_mat_1aa5d20fc86d41d59e4d71ae93daee9726>`<double>(i, 1) = pts[i].y;
	    }
	
	    //Perform PCA analysis
	    :ref:`PCA <doxid-d3/d39/classcv_1_1_p_c_a>` pca_analysis(data_pts, :ref:`Mat <doxid-db/de6/classcv_1_1_mat>`(), :ref:`CV_PCA_DATA_AS_ROW <doxid-d2/df8/group__core__c_1ga8010c4e67ca59cfd17d7a965cea7d198>`);
	
	    //Store the center of the object
	    Point cntr = :ref:`Point <doxid-dc/d84/group__core__basic_1ga1e83eafb2d26b3c93f09e8338bcab192>`(static_cast<int>(pca_analysis.mean.at<double>(0, 0)),
	                      static_cast<int>(pca_analysis.mean.at<double>(0, 1)));
	
	    //Store the eigenvalues and eigenvectors
	    vector<Point2d> eigen_vecs(2);
	    vector<double> eigen_val(2);
	    for (int i = 0; i < 2; ++i)
	    {
	        eigen_vecs[i] = :ref:`Point2d <doxid-dc/d84/group__core__basic_1ga32986d8c6fd05949688352abd112509f>`(pca_analysis.eigenvectors.at<double>(i, 0),
	                                pca_analysis.eigenvectors.at<double>(i, 1));
	
	        eigen_val[i] = pca_analysis.eigenvalues.at<double>(0, i);
	    }
	
	    // Draw the principal components
	    :ref:`circle <doxid-d6/d6e/group__imgproc__draw_1gaf10604b069374903dbd0f0488cb43670>`(img, cntr, 3, :ref:`Scalar <doxid-dc/d84/group__core__basic_1ga599fe92e910c027be274233eccad7beb>`(255, 0, 255), 2);
	    Point p1 = cntr + 0.02 * :ref:`Point <doxid-dc/d84/group__core__basic_1ga1e83eafb2d26b3c93f09e8338bcab192>`(static_cast<int>(eigen_vecs[0].x * eigen_val[0]), static_cast<int>(eigen_vecs[0].y * eigen_val[0]));
	    Point p2 = cntr - 0.02 * :ref:`Point <doxid-dc/d84/group__core__basic_1ga1e83eafb2d26b3c93f09e8338bcab192>`(static_cast<int>(eigen_vecs[1].x * eigen_val[1]), static_cast<int>(eigen_vecs[1].y * eigen_val[1]));
	    drawAxis(img, cntr, p1, :ref:`Scalar <doxid-dc/d84/group__core__basic_1ga599fe92e910c027be274233eccad7beb>`(0, 255, 0), 1);
	    drawAxis(img, cntr, p2, :ref:`Scalar <doxid-dc/d84/group__core__basic_1ga599fe92e910c027be274233eccad7beb>`(255, 255, 0), 5);
	
	    double angle = :ref:`atan2 <doxid-df/dfc/group__cudev_1ga1096ba687de70142e095cc791a8bcd65>`(eigen_vecs[0].y, eigen_vecs[0].x); // orientation in radians
	
	    return angle;
	}

#. **Extract orientation**
   
   Orientation is extracted by the call of getOrientation() function, which performs all the PCA procedure.
   
   .. ref-code-block:: cpp
   
   	//Construct a buffer used by the pca analysis
   	int sz = static_cast<int>(pts.size());
   	Mat data_pts = Mat(sz, 2, :ref:`CV_64FC1 <doxid-d1/d1b/group__core__hal__interface_1ga3b256d68c6002b6afd60ab9538d717a8>`);
   	for (int i = 0; i < data_pts.rows; ++i)
   	{
   	    data_pts.at<double>(i, 0) = pts[i].x;
   	    data_pts.at<double>(i, 1) = pts[i].y;
   	}
   
   	//Perform PCA analysis
   	PCA pca_analysis(data_pts, Mat(), :ref:`CV_PCA_DATA_AS_ROW <doxid-d2/df8/group__core__c_1ga8010c4e67ca59cfd17d7a965cea7d198>`);
   
   	//Store the center of the object
   	Point cntr = :ref:`Point <doxid-dc/d84/group__core__basic_1ga1e83eafb2d26b3c93f09e8338bcab192>`(static_cast<int>(pca_analysis.mean.at<double>(0, 0)),
   	                  static_cast<int>(pca_analysis.mean.at<double>(0, 1)));
   
   	//Store the eigenvalues and eigenvectors
   	vector<Point2d> eigen_vecs(2);
   	vector<double> eigen_val(2);
   	for (int i = 0; i < 2; ++i)
   	{
   	    eigen_vecs[i] = :ref:`Point2d <doxid-dc/d84/group__core__basic_1ga32986d8c6fd05949688352abd112509f>`(pca_analysis.eigenvectors.at<double>(i, 0),
   	                            pca_analysis.eigenvectors.at<double>(i, 1));
   
   	    eigen_val[i] = pca_analysis.eigenvalues.at<double>(0, i);
   	}
   
   First the data need to be arranged in a matrix with size n x 2, where n is the number of data points we have. Then we can perform that PCA analysis. The calculated mean (i.e. center of mass) is stored in the *cntr* variable and the eigenvectors and eigenvalues are stored in the corresponding std::vector’s.
