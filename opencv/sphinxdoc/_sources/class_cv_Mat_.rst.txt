.. _details-d9/de0/classcv_1_1_mat__:

Detailed Documentation
~~~~~~~~~~~~~~~~~~~~~~

Template matrix class derived from :ref:`Mat <doxid-db/de6/classcv_1_1_mat>`.

.. ref-code-block:: cpp

	template<typename _Tp> class Mat_ : public Mat
	{
	public:
	    // ... some specific methods
	    //         and
	    // no new extra fields
	};

The class ``:ref:`Mat_ <doxid-d9/de0/classcv_1_1_mat__>`<_Tp>`` is a *thin* template wrapper on top of the :ref:`Mat <doxid-db/de6/classcv_1_1_mat>` class. It does not have any extra data fields. Nor this class nor :ref:`Mat <doxid-db/de6/classcv_1_1_mat>` has any virtual methods. Thus, references or pointers to these two classes can be freely but carefully converted one to another. For example:

.. ref-code-block:: cpp

	// create a 100x100 8-bit matrix
	Mat M(100,100,CV_8U);
	// this will be compiled fine. no any data conversion will be done.
	Mat_<float>& M1 = (Mat_<float>&)M;
	// the program is likely to crash at the statement below
	M1(99,99) = 1.f;

While :ref:`Mat <doxid-db/de6/classcv_1_1_mat>` is sufficient in most cases, :ref:`Mat\_ <doxid-d9/de0/classcv_1_1_mat__>` can be more convenient if you use a lot of element access operations and if you know matrix type at the compilation time. Note that ``:ref:`Mat::at(int y,int x) <doxid-db/de6/classcv_1_1_mat_1ac40425b84f60b39fd35e03814122661f>``` and ``:ref:`Mat_::operator()(int y,int x) <doxid-d9/de0/classcv_1_1_mat___1af06d3170cbd180f48150241012610bb8>``` do absolutely the same and run at the same speed, but the latter is certainly shorter:

.. ref-code-block:: cpp

	Mat_<double> M(20,20);
	for(int i = 0; i < M.rows; i++)
	    for(int j = 0; j < M.cols; j++)
	        M(i,j) = 1./(i+j+1);
	Mat E, V;
	eigen(M,E,V);
	cout << E.at<double>(0,0)/E.at<double>(M.rows-1,0);

To use :ref:`Mat\_ <doxid-d9/de0/classcv_1_1_mat__>` for multi-channel images/matrices, pass :ref:`Vec <doxid-db/d93/classcv_1_1_vec>` as a :ref:`Mat\_ <doxid-d9/de0/classcv_1_1_mat__>` parameter:

.. ref-code-block:: cpp

	// allocate a 320x240 color image and fill it with green (in RGB space)
	Mat_<Vec3b> img(240, 320, Vec3b(0,255,0));
	// now draw a diagonal white line
	for(int i = 0; i < 100; i++)
	    img(i,i)=Vec3b(255,255,255);
	// and now scramble the 2nd (red) channel of each pixel
	for(int i = 0; i < img.rows; i++)
	    for(int j = 0; j < img.cols; j++)
	        img(i,j)[2] ^= (uchar)(i ^ j);
