.. _details-db/d93/classcv_1_1_vec:

Detailed Documentation
~~~~~~~~~~~~~~~~~~~~~~

Template class for short numerical vectors, a partial case of :ref:`Matx <doxid-db/d15/classcv_1_1_matx>`.

This template class represents short numerical vectors (of 1, 2, 3, 4 ... elements) on which you can perform basic arithmetical operations, access individual elements using [] operator etc. The vectors are allocated on stack, as opposite to std::valarray, std::vector, :ref:`cv::Mat <doxid-db/de6/classcv_1_1_mat>` etc., which elements are dynamically allocated in the heap.

The template takes 2 parameters: In addition to the universal notation like Vec<float, 3>, you can use shorter aliases for the most popular specialized variants of :ref:`Vec <doxid-db/d93/classcv_1_1_vec>`, e.g. Vec3f ~ Vec<float, 3>.

It is possible to convert :ref:`Vec <doxid-db/d93/classcv_1_1_vec>` <T,2> to/from :ref:`Point\_ <doxid-d9/d87/classcv_1_1_point__>`, :ref:`Vec <doxid-db/d93/classcv_1_1_vec>` <T,3> to/from :ref:`Point3\_ <doxid-d4/d29/classcv_1_1_point3__>`, and :ref:`Vec <doxid-db/d93/classcv_1_1_vec>` <T,4> to :ref:`CvScalar <doxid-de/db0/struct_cv_scalar>` or :ref:`Scalar\_ <doxid-d7/d13/classcv_1_1_scalar__>`. Use operator[] to access the elements of :ref:`Vec <doxid-db/d93/classcv_1_1_vec>`.

All the expected vector operations are also implemented:

* v1 = v2 + v3

* v1 = v2 - v3

* v1 = v2 \* scale

* v1 = scale \* v2

* v1 = -v2

* v1 += v2 and other augmenting operations

* v1 == v2, v1 != v2

* norm(v1) (euclidean norm) The :ref:`Vec <doxid-db/d93/classcv_1_1_vec>` class is commonly used to describe pixel types of multi-channel arrays. See :ref:`Mat <doxid-db/de6/classcv_1_1_mat>` for details.
