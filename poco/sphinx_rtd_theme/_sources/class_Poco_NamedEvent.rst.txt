Detailed Documentation
~~~~~~~~~~~~~~~~~~~~~~

An :ref:`NamedEvent <doxid-class_poco_1_1_named_event>` is a global synchronization object that allows one process or thread to signal an other process or thread that a certain event has happened.

Unlike an :ref:`Event <doxid-class_poco_1_1_event>`, which itself is the unit of synchronization, a :ref:`NamedEvent <doxid-class_poco_1_1_named_event>` refers to a named operating system resource being the unit of synchronization. In other words, there can be multiple instances of :ref:`NamedEvent <doxid-class_poco_1_1_named_event>` referring to the same actual synchronization object.

NamedEvents are always autoresetting.

There should not be more than one instance of :ref:`NamedEvent <doxid-class_poco_1_1_named_event>` for a given name in a process. Otherwise, the instances may interfere with each other.
