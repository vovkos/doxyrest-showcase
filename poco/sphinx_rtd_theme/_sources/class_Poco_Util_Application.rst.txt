Detailed Documentation
~~~~~~~~~~~~~~~~~~~~~~

The :ref:`Application <doxid-class_poco_1_1_util_1_1_application>` class implements the main subsystem in a process.

The application class is responsible for initializing all its subsystems.

Subclasses can and should override the following virtual methods:

.. ref-code-block:: cpp

	- initialize() (the one-argument, protected variant)
	- uninitialize()
	- reinitialize()
	- defineOptions()
	- handleOption()
	- main()

The application's main logic should be implemented in the :ref:`main() <doxid-class_poco_1_1_util_1_1_application_1acb65f34a4ebed4fa8b9987904a2555fc>` method.

There may be at most one instance of the :ref:`Application <doxid-class_poco_1_1_util_1_1_application>` class in a process.

The :ref:`Application <doxid-class_poco_1_1_util_1_1_application>` class maintains a :ref:`LayeredConfiguration <doxid-class_poco_1_1_util_1_1_layered_configuration>` (available via the :ref:`config() <doxid-class_poco_1_1_util_1_1_application_1a4c619b854bdce613aec5fd6a991fb529>` member function) consisting of:

.. ref-code-block:: cpp

	- a MapConfiguration (priority -100) storing application-specific
	  properties, as well as properties from bound command line arguments.
	- a SystemConfiguration (priority 100)
	- the configurations loaded with loadConfiguration().

The :ref:`Application <doxid-class_poco_1_1_util_1_1_application>` class sets a few default properties in its configuration. These are:

.. ref-code-block:: cpp

	- application.path: the absolute path to application executable
	- application.name: the file name of the application executable
	- application.baseName: the file name (excluding extension) of the application executable
	- application.dir: the path to the directory where the application executable resides
	- application.configDir: the path to the directory where the last configuration file loaded with loadConfiguration() was found.

If :ref:`loadConfiguration() <doxid-class_poco_1_1_util_1_1_application_1a11e1d403f05c0f980d24f9555c8777b9>` has never been called, application.configDir will be equal to application.dir.

The POCO_APP_MAIN macro can be used to implement main(argc, argv). If POCO has been built with POCO_WIN32_UTF8, POCO_APP_MAIN supports :ref:`Unicode <doxid-class_poco_1_1_unicode>` command line arguments.

	int
	loadConfiguration(int priority = :ref:`PRIO_DEFAULT<doxid-class_poco_1_1_util_1_1_application_1ad2e6b943e41dace26e409449c8ef14bfae77af34ccfbd0fc8247a1c458a811313>`)

Loads configuration information from a default location.

The configuration(s) will be added to the application's :ref:`LayeredConfiguration <doxid-class_poco_1_1_util_1_1_layered_configuration>` with the given priority.

The configuration file(s) must be located in the same directory as the executable or a parent directory of it, and must have the same base name as the executable, with one of the following extensions: .properties, .ini or .xml.

The .properties file, if it exists, is loaded first, followed by the .ini file and the .xml file.

If the application is built in debug mode (the _DEBUG preprocessor macro is defined) and the base name of the appication executable ends with a 'd', a config file without the 'd' ending its base name is also found.

Example: Given the application "SampleAppd.exe", built in debug mode. Then :ref:`loadConfiguration() <doxid-class_poco_1_1_util_1_1_application_1a11e1d403f05c0f980d24f9555c8777b9>` will automatically find a configuration file named "SampleApp.properties" if it exists and if "SampleAppd.properties" cannot be found.

Returns the number of configuration files loaded, which may be zero.

This method must not be called before init(argc, argv) has been called.

Loads configuration information from the file specified by the given path.

The file type is determined by the file extension. The following extensions are supported:

.. ref-code-block:: cpp

	- .properties - properties file (PropertyFileConfiguration)
	- .ini        - initialization file (IniFileConfiguration)
	- .xml        - XML file (XMLConfiguration)

Extensions are not case sensitive.

The configuration will be added to the application's :ref:`LayeredConfiguration <doxid-class_poco_1_1_util_1_1_layered_configuration>` with the given priority.

	virtual
	int
	run()

Runs the application by performing additional (un)initializations and calling the :ref:`main() <doxid-class_poco_1_1_util_1_1_application_1acb65f34a4ebed4fa8b9987904a2555fc>` method.

First calls :ref:`initialize() <doxid-class_poco_1_1_util_1_1_application_1a82f62471a68493049a7d3955f34976ca>`, then calls :ref:`main() <doxid-class_poco_1_1_util_1_1_application_1acb65f34a4ebed4fa8b9987904a2555fc>`, and finally calls :ref:`uninitialize() <doxid-class_poco_1_1_util_1_1_application_1ae441fc171418cf80cc88922cae09ce8f>`. The latter will be called even if :ref:`main() <doxid-class_poco_1_1_util_1_1_application_1acb65f34a4ebed4fa8b9987904a2555fc>` throws an exception. If :ref:`initialize() <doxid-class_poco_1_1_util_1_1_application_1a82f62471a68493049a7d3955f34976ca>` throws an exception, :ref:`main() <doxid-class_poco_1_1_util_1_1_application_1acb65f34a4ebed4fa8b9987904a2555fc>` will not be called and the exception will be propagated to the caller. If :ref:`uninitialize() <doxid-class_poco_1_1_util_1_1_application_1ae441fc171418cf80cc88922cae09ce8f>` throws an exception, the exception will be propagated to the caller.

	void
	stopOptionsProcessing()

If called from an option callback, stops all further options processing.

If called, the following options on the command line will not be processed, and required options will not be checked.

This is useful, for example, if an option for displaying help information has been encountered and no other things besides displaying help shall be done.

.. index:: pair: function; findFile

.. ref-code-block:: cpp
	:class: doxyrest-title-code-block

	bool
	findFile(:ref:`Poco::Path<doxid-class_poco_1_1_path>`& path) const

Searches for the file in path in the application directory.

If path is absolute, the method immediately returns true and leaves path unchanged.

If path is relative, searches for the file in the application directory and in all subsequent parent directories. Returns true and stores the absolute path to the file in path if the file could be found. Returns false and leaves path unchanged otherwise.
