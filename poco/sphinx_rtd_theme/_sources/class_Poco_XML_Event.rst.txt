Detailed Documentation
~~~~~~~~~~~~~~~~~~~~~~

The :ref:`Event <doxid-class_poco_1_1_x_m_l_1_1_event>` interface is used to provide contextual information about an event to the handler processing the event.

An object which implements the :ref:`Event <doxid-class_poco_1_1_x_m_l_1_1_event>` interface is generally passed as the first parameter to an event handler. More specific context information is passed to event handlers by deriving additional interfaces from :ref:`Event <doxid-class_poco_1_1_x_m_l_1_1_event>` which contain information directly relating to the type of event they accompany. These derived interfaces are also implemented by the object passed to the event listener.

	Poco::UInt64
	timeStamp() const

Used to specify the time (in milliseconds relative to the epoch) at which the event was created.

Due to the fact that some systems may not provide this information the value of timeStamp may be not available for all events. When not available, a value of 0 will be returned. Examples of epoch time are the time of the system start or 0:0:0 UTC 1st January 1970. This implementation always returns 0.

.. index:: pair: function; stopPropagation
.. _doxid-class_poco_1_1_x_m_l_1_1_event_1a149de1634a27306411486cb70199d582:

.. ref-code-block:: cpp
	:class: doxyrest-title-code-block

	void
	stopPropagation()

The stopPropagation method is used prevent further propagation of an event during event flow.

If this method is called by any :ref:`EventListener <doxid-class_poco_1_1_x_m_l_1_1_event_listener>` the event will cease propagating through the tree. The event will complete dispatch to all listeners on the current :ref:`EventTarget <doxid-class_poco_1_1_x_m_l_1_1_event_target>` before event flow stops. This method may be used during any stage of event flow.

.. index:: pair: function; preventDefault
.. _doxid-class_poco_1_1_x_m_l_1_1_event_1a7bb70080eddd05adbabf1d5d67357e36:

.. ref-code-block:: cpp
	:class: doxyrest-title-code-block

	void
	preventDefault()

If an event is cancelable, the preventDefault method is used to signify that the event is to be canceled, meaning any default action normally taken by the implementation as a result of the event will not occur.

If, during any stage of event flow, the preventDefault method is called the event is canceled. :ref:`Any <doxid-class_poco_1_1_any>` default action associated with the event will not occur. Calling this method for a non-cancelable event has no effect. Once preventDefault has been called it will remain in effect throughout the remainder of the event's propagation. This method may be used during any stage of event flow.
