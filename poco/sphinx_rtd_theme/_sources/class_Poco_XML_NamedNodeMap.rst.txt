Detailed Documentation
~~~~~~~~~~~~~~~~~~~~~~

Objects implementing the :ref:`NamedNodeMap <doxid-class_poco_1_1_x_m_l_1_1_named_node_map>` interface are used to represent collections of nodes that can be accessed by name.

Note that :ref:`NamedNodeMap <doxid-class_poco_1_1_x_m_l_1_1_named_node_map>` does not inherit from :ref:`NodeList <doxid-class_poco_1_1_x_m_l_1_1_node_list>`; NamedNodeMaps are not maintained in any particular order. Objects contained in an object implementing :ref:`NamedNodeMap <doxid-class_poco_1_1_x_m_l_1_1_named_node_map>` may also be accessed by an ordinal index, but this is simply to allow convenient enumeration of the contents of a :ref:`NamedNodeMap <doxid-class_poco_1_1_x_m_l_1_1_named_node_map>`, and does not imply that the DOM specifies an order to these Nodes.

:ref:`NamedNodeMap <doxid-class_poco_1_1_x_m_l_1_1_named_node_map>` objects in the DOM are live.

A :ref:`NamedNodeMap <doxid-class_poco_1_1_x_m_l_1_1_named_node_map>` returned from a method must be released with a call to :ref:`release() <doxid-class_poco_1_1_x_m_l_1_1_d_o_m_object_1a59fcc225a5bcf0fcf7e49d06ecba249a>` when no longer needed.

Adds a node using its nodeName attribute.

If a node with that name is already present in this map, it is replaced by the new one. As the nodeName attribute is used to derive the name which the node must be stored under, multiple nodes of certain types (those that have a "special" string value) cannot be stored as the names would clash. This is seen as preferable to allowing nodes to be aliased.
