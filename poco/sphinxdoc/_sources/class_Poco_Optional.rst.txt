Detailed Documentation
~~~~~~~~~~~~~~~~~~~~~~

:ref:`Optional <doxid-class_poco_1_1_optional>` is a simple wrapper class for value types that allows to introduce a specified/unspecified state to value objects.

An :ref:`Optional <doxid-class_poco_1_1_optional>` can be default constructed. In this case, the :ref:`Optional <doxid-class_poco_1_1_optional>` will have a Null value and :ref:`isSpecified() <doxid-class_poco_1_1_optional_1aa976774a4bcc0efe46d7abb7e970b40a>` will return false. Calling :ref:`value() <doxid-class_poco_1_1_optional_1a69bcb485f299bfb15877c04fc28c5ce1>` (without default value) on a Null object will throw a NullValueException.

An :ref:`Optional <doxid-class_poco_1_1_optional>` can also be constructed from a value. It is possible to assign a value to an :ref:`Optional <doxid-class_poco_1_1_optional>`, and to reset an :ref:`Optional <doxid-class_poco_1_1_optional>` to contain a Null value by calling :ref:`clear() <doxid-class_poco_1_1_optional_1a66683a23049ac6758ba3a4803c07f32a>`.

For use with :ref:`Optional <doxid-class_poco_1_1_optional>`, the value type should support default construction.

Note that the :ref:`Optional <doxid-class_poco_1_1_optional>` class is basically the same as :ref:`Nullable <doxid-class_poco_1_1_nullable>`. However, serializers may treat :ref:`Nullable <doxid-class_poco_1_1_nullable>` and :ref:`Optional <doxid-class_poco_1_1_optional>` differently. An example is :ref:`XML <doxid-namespace_poco_1_1_x_m_l>` serialization based on :ref:`XML <doxid-namespace_poco_1_1_x_m_l>` Schema, where :ref:`Optional <doxid-class_poco_1_1_optional>` would be used for an element with minOccurs == 0, whereas :ref:`Nullable <doxid-class_poco_1_1_nullable>` would be used on an element with nillable == true.
