Detailed Documentation
~~~~~~~~~~~~~~~~~~~~~~

The :ref:`TemporaryFile <doxid-class_poco_1_1_temporary_file>` class helps with the handling of temporary files.

A unique name for the temporary file is automatically chosen and the file is placed in the directory reserved for temporary files (see :ref:`Path::temp() <doxid-class_poco_1_1_path_1a741af3dc6b4a6b2791ec94a73df89176>`). Obtain the path by calling the :ref:`path() <doxid-class_poco_1_1_file_1aa50604d122698337ab3c1aadc41b3b08>` method (inherited from :ref:`File <doxid-class_poco_1_1_file>`).

The :ref:`TemporaryFile <doxid-class_poco_1_1_temporary_file>` class does not actually create the file - this is up to the application. The class does, however, delete the temporary file - either in the destructor, or immediately before the application terminates.
