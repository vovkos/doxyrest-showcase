Detailed Documentation
~~~~~~~~~~~~~~~~~~~~~~

A Uniform Resource Identifier, as specified in RFC 3986.

The :ref:`URI <doxid-class_poco_1_1_u_r_i>` class provides methods for building URIs from their parts, as well as for splitting URIs into their parts. Furthermore, the class provides methods for resolving relative URIs against base URIs.

The class automatically performs a few normalizations on all URIs and :ref:`URI <doxid-class_poco_1_1_u_r_i>` parts passed to it:

.. ref-code-block:: cpp

	* scheme identifiers are converted to lower case
	* percent-encoded characters are decoded (except for the query string)
	* optionally, dot segments are removed from paths (see normalize())

Note that dealing with query strings requires some precautions, as, internally, query strings are stored in percent-encoded form, while all other parts of the :ref:`URI <doxid-class_poco_1_1_u_r_i>` are stored in decoded form. While parsing query strings from properly encoded URLs generally works, explicitly setting query strings with :ref:`setQuery() <doxid-class_poco_1_1_u_r_i_1a62b1e5651b85076037eebc33329a2a9b>` or extracting query strings with :ref:`getQuery() <doxid-class_poco_1_1_u_r_i_1afabaf036fd5e86ffd8fc3606c9b76170>` may lead to ambiguities. See the descriptions of :ref:`setQuery() <doxid-class_poco_1_1_u_r_i_1a62b1e5651b85076037eebc33329a2a9b>`, :ref:`setRawQuery() <doxid-class_poco_1_1_u_r_i_1a29d545bafe2cfda88c5b7d208c93f34f>`, :ref:`getQuery() <doxid-class_poco_1_1_u_r_i_1afabaf036fd5e86ffd8fc3606c9b76170>` and :ref:`getRawQuery() <doxid-class_poco_1_1_u_r_i_1aa8e72eda40a52d757e2d9993c35debd1>` for more information.

In such a case it's better to use :ref:`getRawQuery() <doxid-class_poco_1_1_u_r_i_1aa8e72eda40a52d757e2d9993c35debd1>` or :ref:`getQueryParameters() <doxid-class_poco_1_1_u_r_i_1aa08ed0cf1d48f701cd4d441e040bbca2>`.

	void
	setQuery(:ref:`const<doxid-zconf_8h_1a2c212835823e3c54a8ab6d95c652660e>` std::string& query)

Sets the query part of the :ref:`URI <doxid-class_poco_1_1_u_r_i>`.

The query string will be percent-encoded. If the query already contains percent-encoded characters, these will be double-encoded, which is probably not what's intended by the caller. Furthermore, ampersand ('&') characters in the query will not be encoded. This could lead to ambiguity issues if the query string contains multiple name-value parameters separated by ampersand, and if any name or value also contains an ampersand. In such a case, it's better to use :ref:`setRawQuery() <doxid-class_poco_1_1_u_r_i_1a29d545bafe2cfda88c5b7d208c93f34f>` with a properly percent-encoded query string, or use :ref:`addQueryParameter() <doxid-class_poco_1_1_u_r_i_1aaa53bb327b493cbeba3c143d5b578950>` or :ref:`setQueryParameters() <doxid-class_poco_1_1_u_r_i_1a2a7f92ea4aad902980e90604ce0fc151>`, which take care of appropriate percent encoding of parameter names and values.
