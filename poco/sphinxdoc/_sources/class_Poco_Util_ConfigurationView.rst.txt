Detailed Documentation
~~~~~~~~~~~~~~~~~~~~~~

This configuration implements a "view" into a sub-hierarchy of another configuration.

For example, given a configuration with the following properties:

.. ref-code-block:: cpp

	config.value1
	config.value2
	config.sub.value1
	config.sub.value2

and a :ref:`ConfigurationView <doxid-class_poco_1_1_util_1_1_configuration_view>` with the prefix "config", then the above properties will be available via the view as

.. ref-code-block:: cpp

	value1
	value2
	sub.value1
	sub.value2

A :ref:`ConfigurationView <doxid-class_poco_1_1_util_1_1_configuration_view>` is most useful in combination with a :ref:`LayeredConfiguration <doxid-class_poco_1_1_util_1_1_layered_configuration>`.

If a property is not found in the view, it is searched in the original configuration. Given the above example configuration, the property named "config.value1" will still be found in the view.

The main reason for this is that placeholder expansion (${property}) still works as expected given a :ref:`ConfigurationView <doxid-class_poco_1_1_util_1_1_configuration_view>`.
