.. _details-class_poco_1_1_util_1_1_x_m_l_configuration:

Detailed Documentation
~~~~~~~~~~~~~~~~~~~~~~

This configuration class extracts configuration properties from an :ref:`XML <doxid-namespace_poco_1_1_x_m_l>` document.

An XPath-like syntax for property names is supported to allow full access to the :ref:`XML <doxid-namespace_poco_1_1_x_m_l>` document. :ref:`XML <doxid-namespace_poco_1_1_x_m_l>` namespaces are not supported. The name of the root element of the :ref:`XML <doxid-namespace_poco_1_1_x_m_l>` document is not significant and ignored. Periods in tag names are not supported.

Given the following :ref:`XML <doxid-namespace_poco_1_1_x_m_l>` document as an example:

.. ref-code-block:: cpp

	<config>
	    <prop1>value1</prop1>
	    <prop2>value2</prop2>
	    <prop3>
	       <prop4 attr="value3"/>
	       <prop4 attr="value4"/>
	    </prop3>
	    <prop5 id="first">value5</prop5>
	    <prop5 id="second">value6</prop5>
	</config>

The following property names would be valid and would yield the shown values:

.. ref-code-block:: cpp

	prop1                 -> value1
	prop2                 -> value2
	prop3.prop4           -> (empty string)
	prop3.prop4[@attr]    -> value3
	prop3.prop4[1][@attr] -> value4
	prop5[0]              -> value5
	prop5[1]              -> value6
	prop5[@id=first]      -> value5
	prop5[@id='second']   -> value6

Enumerating attributes is not supported. Calling keys("prop3.prop4") will return an empty range.

As a special feature, the delimiter character used to delimit property names can be changed to something other than period ('.') by passing the desired character to the constructor. This allows working with :ref:`XML <doxid-namespace_poco_1_1_x_m_l>` documents having element names with periods in them.
