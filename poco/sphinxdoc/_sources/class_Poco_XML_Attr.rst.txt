Detailed Documentation
~~~~~~~~~~~~~~~~~~~~~~

The :ref:`Attr <doxid-class_poco_1_1_x_m_l_1_1_attr>` interface represents an attribute in an :ref:`Element <doxid-class_poco_1_1_x_m_l_1_1_element>` object.

Typically the allowable values for the attribute are defined in a document type definition.

:ref:`Attr <doxid-class_poco_1_1_x_m_l_1_1_attr>` objects inherit the :ref:`Node <doxid-class_poco_1_1_x_m_l_1_1_node>` interface, but since they are not actually child nodes of the element they describe, the DOM does not consider them part of the document tree. Thus, the :ref:`Node <doxid-class_poco_1_1_x_m_l_1_1_node>` attributes parentNode, previousSibling, and nextSibling have a null value for :ref:`Attr <doxid-class_poco_1_1_x_m_l_1_1_attr>` objects. The DOM takes the view that attributes are properties of elements rather than having a separate identity from the elements they are associated with; this should make it more efficient to implement such features as default attributes associated with all elements of a given type. Furthermore, :ref:`Attr <doxid-class_poco_1_1_x_m_l_1_1_attr>` nodes may not be immediate children of a :ref:`DocumentFragment <doxid-class_poco_1_1_x_m_l_1_1_document_fragment>`. However, they can be associated with :ref:`Element <doxid-class_poco_1_1_x_m_l_1_1_element>` nodes contained within a :ref:`DocumentFragment <doxid-class_poco_1_1_x_m_l_1_1_document_fragment>`. In short, users and implementors of the DOM need to be aware that :ref:`Attr <doxid-class_poco_1_1_x_m_l_1_1_attr>` nodes have some things in common with other objects inheriting the :ref:`Node <doxid-class_poco_1_1_x_m_l_1_1_node>` interface, but they also are quite distinct.

The attribute's effective value is determined as follows: if this attribute has been explicitly assigned any value, that value is the attribute's effective value; otherwise, if there is a declaration for this attribute, and that declaration includes a default value, then that default value is the attribute's effective value; otherwise, the attribute does not exist on this element in the structure model until it has been explicitly added. Note that the nodeValue attribute on the :ref:`Attr <doxid-class_poco_1_1_x_m_l_1_1_attr>` instance can also be used to retrieve the string version of the attribute's value(s).

In :ref:`XML <doxid-namespace_poco_1_1_x_m_l>`, where the value of an attribute can contain entity references, the child nodes of the :ref:`Attr <doxid-class_poco_1_1_x_m_l_1_1_attr>` node provide a representation in which entity references are not expanded. These child nodes may be either :ref:`Text <doxid-class_poco_1_1_x_m_l_1_1_text>` or :ref:`EntityReference <doxid-class_poco_1_1_x_m_l_1_1_entity_reference>` nodes. Because the attribute type may be unknown, there are no tokenized attribute values.

	bool
	specified() const

If this attribute was explicitly given a value in the original document, this is true; otherwise, it is false.

Note that the implementation is in charge of this attribute, not the user. If the user changes the value of the attribute (even if it ends up having the same value as the default value) then the specified flag is automatically flipped to true. To re-specify the attribute as the default value from the DTD, the user must delete the attribute. The implementation will then make a new attribute available with specified set to false and the default value (if one exists). In summary:

.. ref-code-block:: cpp

	* If the attribute has an assigned value in the document then specified
	  is true, and the value is the assigned value.
	* If the attribute has no assigned value in the document and has a default
	  value in the DTD, then specified is false, and the value is the default
	  value in the DTD.
	* If the attribute has no assigned value in the document and has a value
	  of #IMPLIED in the DTD, then the attribute does not appear in the structure
	  model of the document.
	* If the attribute is not associated to any element (i.e. because it
	  was just created or was obtained from some removal or cloning operation)
	  specified is true.

	namespaceURI() const

Returns the namespace :ref:`URI <doxid-class_poco_1_1_u_r_i>` of the node.

This is not a computed value that is the result of a namespace lookup based on an examination of the namespace declarations in scope. It is merely the namespace :ref:`URI <doxid-class_poco_1_1_u_r_i>` given at creation time.

For nodes of any type other than ELEMENT_NODE and ATTRIBUTE_NODE and nodes created with a DOM Level 1 method, such as createElement from the :ref:`Document <doxid-class_poco_1_1_x_m_l_1_1_document>` interface, this is always the empty string.
