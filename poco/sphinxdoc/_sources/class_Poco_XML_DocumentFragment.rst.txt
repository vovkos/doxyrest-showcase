.. _details-class_poco_1_1_x_m_l_1_1_document_fragment:

Detailed Documentation
~~~~~~~~~~~~~~~~~~~~~~

:ref:`DocumentFragment <doxid-class_poco_1_1_x_m_l_1_1_document_fragment>` is a "lightweight" or "minimal" :ref:`Document <doxid-class_poco_1_1_x_m_l_1_1_document>` object.

It is very common to want to be able to extract a portion of a document's tree or to create a new fragment of a document. Imagine implementing a user command like cut or rearranging a document by moving fragments around. It is desirable to have an object which can hold such fragments and it is quite natural to use a :ref:`Node <doxid-class_poco_1_1_x_m_l_1_1_node>` for this purpose. While it is true that a :ref:`Document <doxid-class_poco_1_1_x_m_l_1_1_document>` object could fulfill this role, a :ref:`Document <doxid-class_poco_1_1_x_m_l_1_1_document>` object can potentially be a heavyweight object, depending on the underlying implementation. What is really needed for this is a very lightweight object. :ref:`DocumentFragment <doxid-class_poco_1_1_x_m_l_1_1_document_fragment>` is such an object.

Furthermore, various operations such as inserting nodes as children of another :ref:`Node <doxid-class_poco_1_1_x_m_l_1_1_node>` may take :ref:`DocumentFragment <doxid-class_poco_1_1_x_m_l_1_1_document_fragment>` objects as arguments; this results in all the child nodes of the :ref:`DocumentFragment <doxid-class_poco_1_1_x_m_l_1_1_document_fragment>` being moved to the child list of this node.

The children of a :ref:`DocumentFragment <doxid-class_poco_1_1_x_m_l_1_1_document_fragment>` node are zero or more nodes representing the tops of any sub-trees defining the structure of the document. :ref:`DocumentFragment <doxid-class_poco_1_1_x_m_l_1_1_document_fragment>` nodes do not need to be well-formed :ref:`XML <doxid-namespace_poco_1_1_x_m_l>` documents (although they do need to follow the rules imposed upon well-formed :ref:`XML <doxid-namespace_poco_1_1_x_m_l>` parsed entities, which can have multiple top nodes). For example, a :ref:`DocumentFragment <doxid-class_poco_1_1_x_m_l_1_1_document_fragment>` might have only one child and that child node could be a :ref:`Text <doxid-class_poco_1_1_x_m_l_1_1_text>` node. Such a structure model represents neither an HTML document nor a well-formed :ref:`XML <doxid-namespace_poco_1_1_x_m_l>` document.

When a :ref:`DocumentFragment <doxid-class_poco_1_1_x_m_l_1_1_document_fragment>` is inserted into a :ref:`Document <doxid-class_poco_1_1_x_m_l_1_1_document>` (or indeed any other :ref:`Node <doxid-class_poco_1_1_x_m_l_1_1_node>` that may take children) the children of the :ref:`DocumentFragment <doxid-class_poco_1_1_x_m_l_1_1_document_fragment>` and not the :ref:`DocumentFragment <doxid-class_poco_1_1_x_m_l_1_1_document_fragment>` itself are inserted into the :ref:`Node <doxid-class_poco_1_1_x_m_l_1_1_node>`. This makes the :ref:`DocumentFragment <doxid-class_poco_1_1_x_m_l_1_1_document_fragment>` very useful when the user wishes to create nodes that are siblings; the :ref:`DocumentFragment <doxid-class_poco_1_1_x_m_l_1_1_document_fragment>` acts as the parent of these nodes so that the user can use the standard methods from the :ref:`Node <doxid-class_poco_1_1_x_m_l_1_1_node>` interface, such as insertBefore and appendChild.
