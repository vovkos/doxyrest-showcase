Detailed Documentation
~~~~~~~~~~~~~~~~~~~~~~

The :ref:`NodeAppender <doxid-class_poco_1_1_x_m_l_1_1_node_appender>` class provides a very fast way to build larger DOM documents.

In the DOM, child nodes are usually appended to a parent node using the :ref:`appendChild() <doxid-class_poco_1_1_x_m_l_1_1_node_appender_1a9995937be5e35f3831e1993fab08348a>` method. For nodes containing more than a few children, this method can be quite slow, due to the way it's implemented, and because of the requirements of the DOM specification.

While the :ref:`NodeAppender <doxid-class_poco_1_1_x_m_l_1_1_node_appender>` is being used on an :ref:`Element <doxid-class_poco_1_1_x_m_l_1_1_element>`, no children-modifying methods of that :ref:`Element <doxid-class_poco_1_1_x_m_l_1_1_element>` must be used.

This class is not part of the DOM specification.

	void
	appendChild(:ref:`Node<doxid-class_poco_1_1_x_m_l_1_1_node>`* newChild)

Appends the node newChild to the end of the list of children of the parent node specified in the constructor.

If the newChild is already in the tree, it is first removed.

NewChild can be a :ref:`DocumentFragment <doxid-class_poco_1_1_x_m_l_1_1_document_fragment>`. In this case, all children of the fragment become children of the parent element.

In order to speed up the function, no DOM events are fired.
